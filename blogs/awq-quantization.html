Understanding AWQ Quantization

April 10th, 2026

A ground-up implementation of AWQ in plain PyTorch, based on Lin et al., 2023.

Why Quantization Hurts?

When you quantize a weight matrix from float32 down to 4-bit integers, you're forcing every weight into one of only 15 discrete levels. The rounding error introduced is proportional to delta — the step size between levels:

δ = max(|W|) / (2bits − 1)

The larger max(|W|) is, the coarser your grid, and the more error each rounded weight carries. AWQ's core observation is that not all weight channels are equally important. Channels corresponding to large input activations dominate the output — so quantization errors there hurt far more than errors in channels with small activations.

The Scaling Identity

The entire method rests on one algebraic fact. For any positive per-channel scale vector s:

W · X = (W · s) · (X / s)

This is exact — no approximation. If you scale column i of W up by s[i] and row i of X down by s[i], the output is unchanged. But now when you quantize W · s, channel i occupies a larger portion of the quantization range, which means the rounding grid is finer relative to its values — less error per weight.

Core Insight

Protect the weight channels that matter most by scaling them up before quantization, then compensating on the input side. The math stays equivalent; the quantization error distribution shifts in your favor.

Setup

Let's fix our tensor conventions before writing any code. We'll use C input channels and N samples:

  • X — shape (C, N): input activations; each row is one channel across all samples
  • W — shape (C_out, C): weight matrix; each column corresponds to one input channel
  • s_vec — shape (C,): one scale per input channel; scales column j of W and row j of X
import torch

C = 4       # input channels
C_out = 8   # output channels
N = 2       # samples

X = torch.randn(C, N)
W = torch.randn(C_out, C)
Y = W @ X   # ground truth

Quantized Matmul with Per-Channel Scaling

Here's the full implementation. The key is that scaling happens before computing delta — that's what changes the quantization grid for each channel independently.

def awq_quantized_matmul(W, X, s_vec, quant_bits=4):
    """
    s_vec: shape (C,) — one scale factor per input channel.
    Equivalent transform: W' = W * s_vec,  X' = X / s_vec
    """
    # s_vec.unsqueeze(0) → (1, C): scales columns of W
    W_scaled = W * s_vec.unsqueeze(0)
    # s_vec.unsqueeze(1) → (C, 1): scales rows of X
    X_scaled = X / s_vec.unsqueeze(1)

    # Per-row quantization of W_scaled
    max_w, _ = W_scaled.abs().max(dim=-1, keepdim=True)
    delta = max_w / (2**quant_bits - 1)
    W_q = delta * (W_scaled / delta).round()

    return W_q @ X_scaled
Why per-row delta?

Each output neuron (row of W) has its own dynamic range. Using a shared delta across all rows would force small-magnitude rows to share grid resolution with large-magnitude ones — wasteful and error-prone. Per-row delta means each output channel uses its full 4-bit range independently.

Finding the Optimal Scale Vector

The search space for s_vec follows directly from AWQ's insight: channels with large input activations are the ones worth protecting. The paper proposes searching over candidate scales of the form:

s[i] = mean(|X[i, :]|)α, α ∈ [0, 1]

When α = 0, s[i] = 1 (no scaling). When α = 1, s[i] = mean(|X[i, :]|) (full activation magnitude). The exponent interpolates between these extremes, and we search for the best α per channel by minimizing output MSE.

Why absolute mean, not raw mean?

Activations can be negative. A channel with values [1, -1, 1, -1] has zero mean but nonzero magnitude — you want to protect it. Taking abs() first ensures the scale reflects true activation strength.

Greedy Sequential Search

Each s_vec[i] only affects column i of W and row i of X, so you can optimize channels one at a time. Crucially, once you've found the optimal scale for channel i, you carry it forward when optimizing channel i+1 — rather than always starting from all-ones.

def find_optimal_svec(W, X, quant_bits=4, steps=100):
    Y = W @ X                         # ground truth output
    C = W.shape[-1]
    s_opt_vect = torch.ones((C,))    # start with no scaling

    for i in range(C):
        X_channel = X[i, :]

        # Candidate scales: mean_act^alpha for alpha in [0, 1]
        mean_act = X_channel.abs().mean()
        mean_act_alpha = mean_act ** torch.linspace(0, 1, steps)

        # Build candidate s_vecs: carry forward previously found optima
        s_candidates = s_opt_vect.unsqueeze(0).expand(steps, -1).detach().clone()
        s_candidates[:, i] = mean_act_alpha

        # Grid search over candidates
        min_mse, s_opt = float('inf'), 1.0
        for s in range(steps):
            Y_q = awq_quantized_matmul(W, X, s_candidates[s], quant_bits)
            mse = ((Y - Y_q) ** 2).mean()
            if mse < min_mse:
                min_mse = mse
                s_opt = mean_act_alpha[s]

        s_opt_vect[i] = s_opt   # carry forward for next channel

    return s_opt_vect
Why carry forward?

If you reset to all-ones for each channel, you're evaluating candidates in an unrealistic context — every other channel has no scaling, which won't be true when you actually use the combined s_vec. Carrying forward previously found optima means each channel is optimized in the context of the scales already committed to.

Putting It Together

# Find optimal scales using calibration data
s_opt = find_optimal_svec(W, X, quant_bits=4, steps=100)

# Baseline: no scaling (s_vec = ones)
Y_q_baseline = awq_quantized_matmul(W, X, torch.ones(C), quant_bits=4)

# AWQ: optimal per-channel scaling
Y_q_awq = awq_quantized_matmul(W, X, s_opt, quant_bits=4)

print(f"MSE Loss:           {((Y - Y_q_baseline)**2).mean()}")
print(f"MSE Loss (Optimal): {((Y - Y_q_awq)**2).mean()}")
print(f"Optimal S_Vec:      {s_opt}")

The improvement here is modest because W and X are random — there's no structure to exploit. In a real pretrained model, certain channels carry dramatically more activation energy than others, and AWQ's gains become much more pronounced, especially at aggressive quantization levels like 3-bit or 4-bit.

Effect of Quantization Bits

Sweeping across bit-widths shows how AWQ's relative advantage grows at lower precision — exactly where it matters most for deployment.

What This Implementation Leaves Out

This tutorial captures the core mechanism of AWQ. The actual paper and production implementations add several things on top:

  • Real calibration data. Instead of random X, you run a small subset of real inputs through the model to measure true activation magnitudes per channel.
  • Layer-by-layer application. The scale search runs independently for every linear layer in the network, not just one matrix.
  • Integer packing. The quantized weights are packed into efficient integer formats (INT4) for actual inference speedup — this tutorial only simulates the quantization numerically.
  • Group quantization. Rather than one delta per row, production AWQ often uses one delta per group of 128 weights, giving finer-grained control over the quantization grid.
Further Reading
